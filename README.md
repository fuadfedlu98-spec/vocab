# Vocab App (Offline, Flutter, Android 12)

Fully offline vocabulary learning app: local SQLite storage, offline TTS,
spaced-repetition quiz, and local (device-scheduled) reminder notifications.
No backend, no cloud, no internet dependency.

Note on scope: the spec's optional "online dictionary search" feature
(section 7) was intentionally **left out** because it wasn't in the
mandatory screen list and the spec says not to guess on unclear/optional
items. Everything else in the spec is implemented.

## Prerequisites

- Flutter SDK (latest stable) installed: https://docs.flutter.dev/get-started/install
- Android Studio or just the Android SDK/command-line tools + an emulator
  or a physical Android 12+ device with USB debugging enabled.

This project's `android/` folder was hand-authored (not generated by
`flutter create`) because no Flutter SDK was available in the environment
that produced this code. Two files Flutter normally generates locally are
**not** included and must be created on your machine:

1. `android/local.properties` — auto-created by Android Studio/Flutter on
   first build. If you need to make it manually:
   ```
   sdk.dir=/path/to/your/Android/sdk
   flutter.sdk=/path/to/your/flutter/sdk
   ```
2. The Gradle wrapper jar (`android/gradle/wrapper/gradle-wrapper.jar`) —
   regenerate it with:
   ```
   cd android && gradle wrapper --gradle-version 8.6
   ```
   (or simply let Android Studio regenerate it on first open).

The launcher icons in `android/app/src/main/res/mipmap-*/ic_launcher.png`
are plain placeholder squares — replace them with real artwork whenever
you're ready (e.g. via `flutter pub run flutter_launcher_icons`).

## Android 12 notification notes

- On first launch the app requests the `POST_NOTIFICATIONS` runtime
  permission and the "schedule exact alarms" permission, both required
  on Android 12+ for reliable reminders.
- Reminders are scheduled with the device's own alarm/notification
  scheduler (`flutter_local_notifications` + `zonedSchedule`), not
  simulated in app code, so they fire even if the app is closed.
- A boot receiver (`ScheduledNotificationBootReceiver`) is registered in
  the manifest so reminders survive a device restart.

## Spaced repetition logic (quiz)

- Each quiz session pulls 5–10 questions, prioritizing words whose
  `next_review_time` has passed, then filling any remaining slots with
  random words from the full set.
- Correct answer → `correct_count` increments, `next_review_time` pushed
  further out (interval grows, capped at 30 days).
- Wrong answer → `wrong_count` increments, `next_review_time` set to
  10 minutes from now so it resurfaces soon.
- Words are never removed from the system regardless of performance.
